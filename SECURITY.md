# Security Policy

## Supported Versions

This repository's active development branch is the primary supported version.

## Reporting a Vulnerability

Please do not open a public issue for sensitive security concerns.

Use GitHub's private security advisory workflow for the repository when available.

If a private advisory is not available, contact the maintainers through a private GitHub channel and include:

- a short summary of the issue
- affected files or commands
- steps to reproduce
- any relevant logs or screenshots

## Response Expectations

Reports should be acknowledged as soon as practical, and confirmed issues will be handled privately before public disclosure.
